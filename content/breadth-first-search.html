Introduction

Breadth-first search is an algorithm for traversing a graph. Given a graph and a source vertex, breadth-first search explores the graph to discover every vertex that is reachable from the source vertex. It proceeds by exploring all of the adjacent vertices at the present depth prior to moving on to the vertices at the next depth level. Breadth-first search can also be used to find the shortest path between two vertices.

Traversal

  1. Initialize an empty queue.
  2. Mark the source vertex as visited and enqueue it.
  3. Dequeue a vertex and consider each of its adjacent vertices.
  4. If a vertex adjacent to the dequeued vertex has not been visited then mark it as visited and enqueue it.
  5. Repeat steps 3-4 until the queue is empty.

Search

  1. Initialize an empty queue.
  2. Mark the source vertex as visited and enqueue it.
  3. Dequeue a vertex and consider each of its adjacent vertices.
  4. If a vertex adjacent to the dequeued vertex has not been visited then check if it is the target vertex. If it is then return successfully. Otherwise, mark it as visited and enqueue it.
  5. Repeat steps 3-4 until the queue is empty.

Shortest Path

Breadth-first search can be used to find the shortest path between two vertices provided all the weights in the graph are one.
